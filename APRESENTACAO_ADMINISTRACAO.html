Data organization

Five areas with distinct responsibilities

1. Commercial

What the customer requested · edits · approvals · cancellations · full audit log

2. Logistics

What physically ships · one delivery per shipment (can hold 20+ SKUs) · line-level status inside each delivery · redeliveries linked to failed lines · stock movements

3. Fiscal

Invoices and credit notes · exactly as emitted in InvoiceXpress · never overwritten, always auditable

4. Financial

Payments received · CN applications to invoices · real-time customer balance

5. Integration

Emission queue for InvoiceXpress · log of every API call · automatic retry

Every business fact (ordered, delivered, invoiced, paid) has its own record and is never overwritten.

OptimoRoute has no line-level status today.

Phase 2: read incidents DB. Phase 3: OptimoRoute line-status if supported.

Invoicing with InvoiceXpress

Three levels of validation before emission

Level 1

Automatic validation · Customer has NIF? Amounts add up? All deliveries closed?

Level 2

Human review · The app shows each ready-to-emit invoice to admin with a checklist before bulk emission.

Level 3

Nightly reconciliation · Every night the system compares Supabase vs InvoiceXpress. If there's a mismatch, finance is alerted.

Daily invoicing (~85%)

Fires when routes close + shipping notes emitted.

Monthly invoicing (~15%)

Month-end (day 1, early morning) → one invoice per customer covering everything delivered in the month.

Outside our control

External dependencies

AreaWhat we needIf not ready
Stock / Warehouse Inbound/outbound records on the same DB or via API Operate blind on stock levels — manual check at warehouse
Procurement Supplier purchases visible so we can cross-check with demand No automatic shortage alert → warned only at picking time
InvoiceXpress 3 technical items: webhook, idempotency (client_reference), due-date formula Known workarounds — mild operational impact

None of these block v2. Each has a contingency plan. But the full value only lands if the integrations happen.
